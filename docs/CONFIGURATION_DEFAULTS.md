# Значения по умолчанию конфигурации

## Обзор

Этот документ содержит полный справочник значений по умолчанию для всех параметров конфигурации в классе `Settings` (`src/config.py`).

## Обязательные параметры

Эти параметры **НЕ имеют** значений по умолчанию и **ДОЛЖНЫ** быть установлены в файле `.env`:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `BOT_TOKEN` | `str` | Токен Telegram бота от @BotFather |
| `ADMIN_TELEGRAM_ID` | `int` | Telegram ID администратора бота |
| `DATABASE_URL` | `str` | URL подключения к базе данных |

## Необязательные параметры

Все параметры ниже имеют разумные значения по умолчанию и могут быть опущены в `.env` файле.

### Окружение

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `ENV` | `str` | `"development"` | Режим окружения: development, test, staging, production |

### Система парсинга

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `PARSING_ENABLED` | `bool` | `False` | Включить автоматический парсинг результатов игр |
| `PARSING_CHECK_INTERVAL` | `int` | `60` | Интервал проверки парсинга в секундах (1 минута) |

### Логирование

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `LOG_LEVEL` | `str` | `"INFO"` | Уровень логирования: DEBUG, INFO, WARNING, ERROR, CRITICAL |
| `LOG_FILE` | `Optional[str]` | `None` | Путь к файлу логов (None = только консоль) |
| `STRUCTURED_LOGGING` | `bool` | `False` | Включить структурированное логирование в формате JSON |

### Пул подключений к БД

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `DB_POOL_SIZE` | `int` | `5` | Максимальное количество постоянных подключений в пуле |
| `DB_MAX_OVERFLOW` | `int` | `10` | Максимальное количество дополнительных подключений сверх пула |
| `DB_POOL_TIMEOUT` | `int` | `30` | Таймаут подключения в секундах |

**Примечание**: Общее максимальное количество подключений = `DB_POOL_SIZE` + `DB_MAX_OVERFLOW` = 15

### Фоновые задачи

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `TASK_CHECK_INTERVAL` | `int` | `300` | Интервал проверки фоновых задач в секундах (5 минут) |
| `BACKUP_ENABLED` | `bool` | `True` | Включить автоматическое резервное копирование БД |
| `BACKUP_INTERVAL` | `int` | `86400` | Интервал резервного копирования в секундах (24 часа) |
| `BACKUP_RETENTION` | `int` | `7` | Количество резервных копий для хранения |

### Безопасность

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `RATE_LIMIT_ENABLED` | `bool` | `True` | Включить ограничение частоты запросов для защиты от спама |
| `RATE_LIMIT_MAX_REQUESTS` | `int` | `20` | Максимальное количество запросов на пользователя в минуту |
| `SESSION_TIMEOUT` | `int` | `3600` | Таймаут сессии в секундах (1 час) |

### Флаги функций

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `SHOP_ENABLED` | `bool` | `True` | Включить систему магазина |
| `GAMES_ENABLED` | `bool` | `True` | Включить игровые команды и отслеживание очков |
| `ACHIEVEMENTS_ENABLED` | `bool` | `True` | Включить систему достижений/значков |
| `SOCIAL_FEATURES_ENABLED` | `bool` | `True` | Включить социальные функции (профили, друзья, подарки) |
| `DND_SYSTEM_ENABLED` | `bool` | `False` | Включить систему D&D (экспериментальная функция) |

### Уведомления

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `ADMIN_NOTIFICATIONS_ENABLED` | `bool` | `True` | Включить уведомления администратора об ошибках |
| `NOTIFICATION_COOLDOWN` | `int` | `300` | Задержка между уведомлениями в секундах (5 минут) |

### Производительность / Кэширование

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `CACHE_ENABLED` | `bool` | `False` | Включить кэширование часто используемых данных |
| `CACHE_BACKEND` | `str` | `"memory"` | Бэкенд кэширования: 'memory' или 'redis' |
| `REDIS_URL` | `Optional[str]` | `None` | URL Redis для кэширования (обязателен если CACHE_BACKEND=redis) |
| `CACHE_TTL` | `int` | `3600` | Время жизни кэша в секундах (1 час) |

### Разработка

| Параметр | Тип | Значение по умолчанию | Описание |
|----------|-----|----------------------|----------|
| `DEBUG` | `bool` | `False` | Включить режим отладки с подробными ошибками (автоматически включается в development) |
| `HOT_RELOAD` | `bool` | `False` | Включить горячую перезагрузку для разработки |
| `TEST_MODE` | `bool` | `False` | Включить тестовый режим с мок-сервисами (автоматически включается в test) |

## Условная валидация

Некоторые параметры имеют условные требования:

### REDIS_URL
- **Условие**: Обязателен, если `CACHE_BACKEND="redis"`
- **Валидация**: Если кэш-бэкенд установлен на Redis, но REDIS_URL не указан, будет выброшена ошибка валидации

### DEBUG
- **Условие**: Автоматически включается в окружении `development`
- **Поведение**: Если `ENV="development"`, DEBUG будет True, если не установлен явно в False

### TEST_MODE
- **Условие**: Автоматически включается в окружении `test`
- **Поведение**: Если `ENV="test"`, TEST_MODE будет True, если не установлен явно в False

## Рекомендации по использованию

### Для локальной разработки
Минимальный `.env` файл:
```env
BOT_TOKEN=your_bot_token_here
ADMIN_TELEGRAM_ID=123456789
DATABASE_URL=sqlite:///data/bot.db
```

Все остальные параметры будут использовать значения по умолчанию, подходящие для разработки.

### Для production
Рекомендуется явно установить:
```env
ENV=production
BOT_TOKEN=your_production_token
ADMIN_TELEGRAM_ID=123456789
DATABASE_URL=postgresql://user:pass@host:5432/db

# Логирование
LOG_LEVEL=WARNING
LOG_FILE=logs/bot.log

# Безопасность
RATE_LIMIT_ENABLED=true
RATE_LIMIT_MAX_REQUESTS=10

# Резервное копирование
BACKUP_ENABLED=true
BACKUP_INTERVAL=43200  # 12 часов

# Кэширование (для высоконагруженных ботов)
CACHE_ENABLED=true
CACHE_BACKEND=redis
REDIS_URL=redis://localhost:6379/0
```

### Для тестирования
```env
ENV=test
BOT_TOKEN=test_token
ADMIN_TELEGRAM_ID=123456789
DATABASE_URL=sqlite:///:memory:
TEST_MODE=true
```

## Обоснование значений по умолчанию

### Почему PARSING_ENABLED=False?
Парсинг - это дополнительная функция, которая требует настройки правил парсинга в БД. Безопаснее начинать с отключенным парсингом и включать его явно после настройки.

### Почему BACKUP_ENABLED=True?
Резервное копирование критично для production окружений. Лучше иметь его включенным по умолчанию, чем потерять данные.

### Почему RATE_LIMIT_ENABLED=True?
Защита от спама и злоупотреблений должна быть включена по умолчанию для безопасности.

### Почему CACHE_ENABLED=False?
Кэширование добавляет сложность и может привести к устаревшим данным. Для небольших ботов оно не требуется, поэтому отключено по умолчанию.

### Почему DND_SYSTEM_ENABLED=False?
Система D&D является экспериментальной и не полностью реализована. Она отключена по умолчанию, чтобы избежать путаницы.

### Почему большинство feature flags=True?
Основные функции (магазин, игры, достижения) являются частью базовой функциональности бота. Они включены по умолчанию для лучшего опыта "из коробки".

## Изменение значений по умолчанию

Если вам нужно изменить значение по умолчанию:

1. Обновите значение в `src/config.py` в соответствующем `Field(default=...)`
2. Обновите эту документацию
3. Обновите `config/.env.example` с новым значением
4. Обновите тесты в `tests/unit/test_config.py`
5. Добавьте запись в CHANGELOG.md с обоснованием изменения
