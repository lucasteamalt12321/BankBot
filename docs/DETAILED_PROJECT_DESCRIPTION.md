# Подробное описание проекта BankBot (LucasTeam)

## 1. Обзор проекта

BankBot — это Telegram-бот-агрегатор, объединяющий несколько игровых платформ в единую экосистему с общей валютой (банковские монеты). Проект разработан для LucasTeam и поддерживает 5 игровых платформ: GD Cards, Shmalala, True Mafia, Bunker RP и D&D Мастерская.

**Ключевые особенности:**
- Единая система балансов для всех игр
- Автоматический парсинг сообщений из разных ботов
- Магазин товаров и услуг
- Система достижений и наград
- Социальные функции (друзья, подарки, кланы)
- Система D&D ролевых игр
- Мини-игры (Города, Убийственные слова, Уровни GD)
- Административные функции и рассылки

**Статус:** Проект находится в активной разработке, реализована миграция на новую систему парсинга (Фаза 0 завершена).

### 2.2 Архитектурные паттерны

**Модульная архитектура:**
- Разделение на логические модули (bot/, core/, database/, utils/, src/)
- Каждый модуль имеет свою ответственность
- Минимальные зависимости между модулями

**Слойная структура:**
```
Пользовательский интерфейс (Telegram API)
    ↓
Обработчики (handlers/)
    ↓
Бизнес-логика (core/managers/, core/systems/)
    ↓
Слой данных (database/, src/)
    ↓
База данных (SQLite)
```

**Паттерны проектирования:**
- **Strategy** — парсеры для разных игр
- **Factory** — создание парсеров через registry
- **Observer** — система уведомлений
- **Repository** — абстракция доступа к данным
- **Middleware** — обработка ошибок и логирование

## 3. Система парсинга

### 3.1 Две системы парсинга

**Старая система (core/parsers/):**
- Классические парсеры для каждой игры
- Простая логика извлечения данных
- Используется для ручного парсинга

**Новая система (src/):**
- Централизованная система обработки сообщений
- Классификатор типов сообщений
- Идемпотентная обработка (предотвращение дублирования)
- Аудиторский логгинг
- Управление балансами через менеджер

### 3.2 Поддерживаемые игры и сообщения

**GD Cards:**
- `ПРОФИЛЬ` + `Орбы:` — профиль игрока
- `🃏 НОВАЯ КАРТА 🃏` — начисление очков

**Shmalala:**
- `🎣 [Рыбалка] 🎣` + `Рыбак:` + `Монеты: +` — рыбалка
- `Лайк! Вы повысили рейтинг пользователя` — карма (+1 всегда)
- `[Самые богатые в этом чате]` — топ по деньгам
- `[Самые крутые по Карме в этом чате]` — топ по карме

**True Mafia:**
- `Игра окончена!` + `Победители:` — окончание игры
- `👤` + `💵 Деньги:` — профиль игрока

**Bunker RP:**
- `Прошли в бункер:` — окончание игры
- `👤` + `💵 Деньги:` — профиль игрока

### 3.3 Коэффициенты конверсии

```json
{
  "GD Cards": 1,
  "Shmalala": 1,
  "Shmalala Karma": 1,
  "True Mafia": 15,
  "Bunker RP": 20
}
```

**Принцип работы:**
- При парсинге профиля: вычисляется дельта баланса в игре
- Дельта умножается на коэффициент и начисляется на банковский баланс
- При парсинге начислений: сумма умножается на коэффициент
- Для True Mafia и Bunker RP: победители получают фиксированную сумму (10 и 30 соответственно)

### 3.4 Идемпотентность

Каждое сообщение получает уникальный ID на основе SHA-256 от текста и времени. Это предотвращает двойное начисление при повторной обработке.

## 4. База данных

### 4.1 Структура БД

**Основные таблицы:**

**users** — пользователи:
- id, telegram_id, username, first_name, last_name
- balance (банковский баланс)
- daily_streak, last_daily
- total_earned, created_at, last_activity
- is_vip, vip_until
- is_admin
- sticker_unlimited, sticker_unlimited_until
- total_purchases

**transactions** — транзакции:
- id, user_id, amount, transaction_type
- source_game, description, meta_data
- created_at

**bot_balances** — балансы в играх:
- id, user_id, game
- last_balance (для профилей)
- current_bot_balance (накопленные начисления)
- created_at, last_updated

**shop_items** — товары магазина:
- id, category_id, name, description
- price, item_type, meta_data
- purchase_limit, cooldown_hours
- is_active

**user_purchases** — покупки:
- id, user_id, item_id
- purchase_price, purchased_at, expires_at
- is_active, meta_data

**game_sessions** — игровые сессии:
- id, game_type, status
- current_player_id, last_city, used_cities
- game_data, created_at, updated_at

**dnd_sessions, dnd_characters, dnd_dice_rolls, dnd_quests** — D&D система

**achievements, user_achievements** — достижения

**friendships, gifts, clans, clan_members** — социальные функции

**user_notifications** — уведомления

**parsing_rules, parsed_transactions** — правила парсинга

### 4.2 Миграции

Система миграций на основе SQL-файлов с нумерацией:
- `001_initial_schema.sql` — начальная схема
- `add_bot_balances_table.py` — добавление таблицы bot_balances
- `update_shop_prices_migration.py` — обновление цен
- Архив старых миграций в `database/migrations/archive/`

### 4.3 Резервное копирование

Система автоматических бэкапов:
- Создание ZIP-архивов с timestamp
- Хранение в папке `backups/`
- Очистка старых бэкапов (по умолчанию 7 дней)
- Восстановление из бэкапа

## 8. Система мини-игр

### 8.1 CitiesGame

**Правила:**
- Города должны начинаться на последнюю букву предыдущего
- Поддержка сложности (easy/medium/hard)
- Награда: 5 монет за правильный ответ

### 8.2 KillerWordsGame

**Правила:**
- Слова с определенными комбинациями букв (убийственные)
- Категории: оружие, яды, стихии, животные
- Награда: 15 монет за убийственное слово
- Штраф: -3 монеты за слабое слово

### 8.3 GDLevelsGame

**Правила:**
- Уровни Geometry Dash
- То же правило, что и в Городах
- Награда: 5 монет

### 8.4 GamesSystem

**Функции:**
- Создание сессий
- Присоединение игроков (максимум 6)
- Запуск/остановка игр
- Обработка ходов
- Начисление наград
