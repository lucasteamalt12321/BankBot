# 🧪 Бета-команды LucasTeam Bank

Этот документ содержит спецификацию 27 новых пользовательских команд, которые планируются к реализации.

## 📋 Статус: В разработке

Все команды доступны для просмотра через `/beta`, но пока не реализованы.

---

## 💰 Экономика и торговля (7 команд)

### 1. `/market`
**Описание:** Рыночная площадка для торговли между игроками

**Функционал:**
- Просмотр активных предложений купли/продажи
- Фильтрация по категориям (предметы, услуги)
- Сортировка по цене, дате
- Поиск по названию

**Пример использования:**
```
/market
→ Показывает список всех активных предложений

/market стикеры
→ Показывает только предложения со стикерами
```

**Требования:**
- Новая таблица `marketplace_listings`
- Поля: id, seller_id, item_type, item_id, price, created_at, expires_at, status
- Автоматическое удаление истекших объявлений (через 7 дней)

---

### 2. `/sell <предмет> <цена>`
**Описание:** Выставить предмет на продажу

**Функционал:**
- Продажа предметов из инвентаря другим игрокам
- Комиссия 5% в пользу банка
- Автоматическое снятие предмета с продажи при покупке
- Возможность отменить объявление

**Пример использования:**
```
/sell стикеры_24ч 150
→ Выставляет "Безлимитные стикеры на 24 часа" за 150 монет

/sell отмена 123
→ Отменяет объявление с ID 123
```

**Требования:**
- Проверка наличия предмета в инвентаре
- Блокировка предмета на время продажи
- Уведомление продавца при покупке

---

### 3. `/trade <@user>`
**Описание:** Предложить обмен пользователю

**Функционал:**
- Безопасный обмен предметами/монетами
- Подтверждение обеими сторонами
- Окно обмена с предпросмотром
- Таймаут 5 минут на подтверждение

**Пример использования:**
```
/trade @john_doe
→ Открывает окно обмена с пользователем
→ Обе стороны добавляют предметы/монеты
→ Обе стороны подтверждают
→ Обмен завершается
```

**Требования:**
- Новая таблица `trade_sessions`
- Атомарность транзакции (все или ничего)
- Защита от мошенничества

---

### 4. `/loan <сумма>`
**Описание:** Взять кредит в банке

**Функционал:**
- Кредиты под проценты (10% в неделю)
- Лимит зависит от репутации пользователя
- Максимум 3 активных кредита
- Автоматическое начисление процентов

**Пример использования:**
```
/loan 500
→ Берет кредит на 500 монет
→ Через неделю нужно вернуть 550 монет
```

**Требования:**
- Новая таблица `loans`
- Поля: id, user_id, amount, interest_rate, issued_at, due_date, status
- Система репутации (credit_score)
- Уведомления о приближающемся сроке

---

### 5. `/repay [сумма]`
**Описание:** Погасить кредит

**Функционал:**
- Частичное или полное погашение
- История кредитов
- Улучшение кредитного рейтинга при своевременной оплате
- Штрафы за просрочку

**Пример использования:**
```
/repay
→ Показывает все активные кредиты

/repay 250
→ Погашает 250 монет от общей суммы долга

/repay all
→ Погашает все кредиты полностью
```

---

### 6. `/invest <сумма> <срок>`
**Описание:** Инвестировать монеты

**Функционал:**
- Депозит с процентами (5-15% в зависимости от срока)
- Сроки: 1 день (5%), 3 дня (8%), 7 дней (12%), 30 дней (15%)
- Досрочное снятие с потерей процентов
- Автоматическое начисление при истечении срока

**Пример использования:**
```
/invest 1000 7d
→ Инвестирует 1000 монет на 7 дней
→ Через 7 дней получит 1120 монет (12% прибыль)
```

**Требования:**
- Новая таблица `investments`
- Фоновая задача для начисления процентов
- Уведомления о завершении депозита

---

### 7. `/portfolio`
**Описание:** Инвестиционный портфель

**Функционал:**
- Активные депозиты и кредиты
- Прогнозируемый доход/расход
- График доходности
- Общая статистика

**Пример использования:**
```
/portfolio
→ Показывает:
   • 2 активных депозита (ожидаемый доход: 240 монет)
   • 1 активный кредит (к оплате: 550 монет)
   • Чистая позиция: -310 монет
```

## 🎯 Квесты и задания (5 команд)

### 8. `/quests`
**Описание:** Доступные квесты

**Функционал:**
- Ежедневные квесты (обновляются каждый день)
- Еженедельные квесты (обновляются каждую неделю)
- Сюжетные квесты (разблокируются по мере прогресса)
- Награды за выполнение

**Типы квестов:**
- Заработать X монет
- Совершить X покупок
- Пригласить X друзей
- Выиграть X игр
- Войти в систему X дней подряд

**Пример использования:**
```
/quests
→ Показывает список доступных квестов с наградами
```

---

### 9. `/quest_start <id>`
**Описание:** Начать квест

**Функционал:**
- Активация квеста
- Отслеживание прогресса
- Максимум 5 активных квестов одновременно

**Пример использования:**
```
/quest_start 42
→ Активирует квест "Заработай 1000 монет"
→ Прогресс: 0/1000
```

---

### 10. `/quest_progress`
**Описание:** Прогресс по активным квестам

**Функционал:**
- Текущие задачи
- Процент выполнения
- Оставшееся время (для временных квестов)

**Пример использования:**
```
/quest_progress
→ Активные квесты:
   1. Заработай 1000 монет (450/1000) - 45%
   2. Войди 7 дней подряд (3/7) - 43%
   3. Купи 5 предметов (2/5) - 40%
```

---

### 11. `/quest_complete <id>`
**Описание:** Завершить квест

**Функционал:**
- Получение награды
- Разблокировка новых квестов
- Обновление статистики

**Пример использования:**
```
/quest_complete 42
→ Квест "Заработай 1000 монет" завершен!
→ Награда: 200 монет + значок "Трудяга"
```

---

### 12. `/quest_history`
**Описание:** История выполненных квестов

**Функционал:**
- Статистика по квестам
- Общие награды

**Пример использования:**
```
/quest_history
→ Выполнено квестов: 47
→ Получено наград: 9400 монет
→ Разблокировано значков: 12
```

## 🔧 Техническая реализация

### Новые таблицы базы данных:

1. **marketplace_listings** - объявления на рынке
2. **trade_sessions** - сессии обмена
3. **loans** - кредиты
4. **investments** - инвестиции
5. **quests** - квесты
6. **user_quests** - прогресс по квестам
7. **tournaments** - турниры
8. **tournament_participants** - участники турниров
9. **user_titles** - титулы пользователей
10. **user_badges** - значки пользователей
11. **user_themes** - темы оформления
12. **events** - события
13. **lottery_tickets** - лотерейные билеты

### Фоновые задачи:

- Начисление процентов по депозитам
- Начисление процентов по кредитам
- Удаление истекших объявлений
- Обновление ежедневных квестов
- Проведение розыгрышей лотереи
- Обновление рейтингов

### API endpoints (для будущего веб-интерфейса):

- `/api/market` - рыночная площадка
- `/api/quests` - квесты
- `/api/leaderboard` - рейтинги
- `/api/tournaments` - турниры
- `/api/analytics` - аналитика
