### Этап 1: Mom Module (приоритет: HIGH, параллельно с веб-разработкой)

**Вес:** 20%  
**Цель:** Веб-приложение «Тренажёр чтения и понимания» для детей с умственной отсталостью (1 класс).

**Порядок реализации:**
1. **MOM-01:** Веб-приложение (статика) (6%)
   - Создать `webapp/reading_trainer/index.html`
   - Два экрана: чтение (6 предложений) и вопросы (2-3 вопроса)
   - Навигация: «Дальше», «Назад», «Новый текст»
   - Кнопки A+/A- для регулировки шрифта (36-48px)
   - Адаптивный дизайн для телефонов/планшетов
   - Крупные кнопки (минимум 44x44px)

2. **MOM-03:** Backend `/reading_generate` (5%)
   - Flask-маршрут в `run_bot.py`
   - Интеграция с HF Inference API (mistralai/Mistral-7B-Instruct или google/flan-t5-base)
   - Промпт: генерация 6 предложений (3-4 слова) + 2-3 вопроса в JSON
   - Резервные наборы (reading_fallbacks.json) при ошибках API
   - LRU-кэш последних 20 результатов
   - Таймаут 5 секунд

3. **MOM-04:** Фронтенд-логика (5%)
   - Загрузка текста через fetch()
   - Переключение между экранами (чтение ↔ вопросы)
   - Проверка ответов (регистронезависимое сравнение)
   - Хранение прогресса (текущий вопрос, размер шрифта в localStorage)
   - Реализация печати: единый лист с предложениями + вопросы с пустыми строками
   - @media print для скрытия кнопок и полей ввода

4. **MOM-02:** Хостинг и команда в боте (3%)
   - Размещение статики в `webapp/reading_trainer/`
   - Настройка `vercel.json` для маршрута `/reading_trainer`
   - Команда `/reading_trainer` в боте со ссылкой и inline-кнопкой

5. **MOM-05:** Дополнительные улучшения (1%)
   - Озвучивание предложений (Web Speech API, опционально)
   - Кнопка «Подсказка» (первая буква ответа)
   - Сохранение статистики для мамы (опционально)

**Критерий завершения:** 
- Ребёнок может прочитать 6 предложений, ответить на вопросы, вернуться к чтению
- Печать работает: один лист с предложениями и вопросами с пустыми строками
- Генерация через HF работает, при ошибках используется резерв
- Корректное отображение на телефонах (iPhone SE, Android 10+)

**Оценка времени:** 6-8 часов

### Этап 2: GD Core (приоритет: HIGH)

**Вес:** 30%  
**Цель:** Внутренний топ прохождений LucasTeam.

**Порядок реализации:**
1. **GD-02:** `/submit` (4%)
   - Обработчик для видео/фото
   - Создание заявки в `submissions`
   - Проверка дубликатов

2. **GD-03:** `/moderate` (5%)
   - Админ-панель с пагинацией
   - Inline-кнопки: ✅ Подтвердить, ❌ Отклонить
   - Ввод названия уровня
   - Обновление `player_stats`, `level_completions`

3. **GD-04:** Логика сложности (4%)
   - Функция `is_level_eligible(level_id, user_id)`
   - Проверка хардеста и топ-100
   - Интеграция в модерацию

4. **GD-05:** Статистика (5%)
   - `/leaderboard` — топ-100 уровней
   - `/my_stats` — личная статистика
   - `/player_stats @user` — статистика игрока

5. **GD-06:** Админ-команды (4%)
   - `/add_level <название> <позиция> [ссылка]`
   - `/set_level_position <название> <новая_позиция>`
   - Атомарное обновление позиций

6. **GD-07:** GD API (3%)
   - Установить `gd.py`
   - `/gd_user <ник>` — статистика игрока
   - `/gd_level <id>` — информация об уровне

**Критерий завершения:** Полный цикл: заявка → модерация → статистика. Админ может управлять уровнями.

**Оценка времени:** 8-12 часов

### Этап 4: AI Manager (приоритет: HIGH)

**Вес:** 15%  
**Цель:** Единый интерфейс для нескольких ИИ-моделей с автопереключением.

**Порядок реализации:**
1. **AI-01:** Менеджер моделей (5%)
   - Класс `AIModelManager` в `bot/ai/model_manager.py`
   - Конфигурация из env: `AI_PROVIDERS` (JSON)
   - Поддержка HF, OpenRouter, Ollama
   - Автопереключение при ошибках (429, 403, 500)
   - Кэширование ответов (5 минут)

2. **AI-02:** `/chat <персонаж> <текст>` (3%)
   - Промпты для олеговируса и чая
   - Вызов `ai_manager.get_response(prompt)`

3. **AI-03:** `/generate_prayer` (3%)
   - Промпт для генерации молитвы
   - Таймаут 10 секунд

4. **AI-04:** `/ask_canon <вопрос>` (2%)
   - База знаний `data/canon_knowledge.txt`
   - Поиск по ключевым словам или через ИИ

5. **AI-05:** `/ai_model <название>` (2%)
   - Сохранение выбора в `user_preferences`
   - Использование при вызовах команд

**Критерий завершения:** AI Manager работает, переключается между провайдерами, все команды функционируют.

**Оценка времени:** 6-8 часов

## Критерии качества

- Все внешние API-вызовы имеют таймауты (5 сек) и retry (2 раза)
- Логирование: INFO для успешных операций, ERROR для ошибок
- Безопасность: секреты в env, никаких хардкодов
- Тестирование: ручное тестирование каждого deliverable
- Ruff: 0 ошибок после каждого модуля
- Memory Bank: обновление после каждого завершённого deliverable

## Риски и зависимости

**Риски:**
- Lichess API может быть недоступен → добавить fallback
- HF Inference API может иметь лимиты → автопереключение на Ollama
- GD API (`gd.py`) может требовать прокси → обработка SSL-ошибок
- Mini App может не открываться в старых версиях Telegram → тестирование на разных устройствах

**Зависимости:**
- Mom Module зависит от AI Manager (MOM-04 ← AI-01)
- Universe UN-03 зависит от AI Manager (UN-03 ← AI-01)
- Chess CH-05 зависит от таблицы `user_coins` (CH-05 ← Этап 0)

**Рекомендация:** Начать с Этапа 0 (БД), затем параллельно AI Manager (для Mom и Universe) и Mom Module (веб-разработка).
