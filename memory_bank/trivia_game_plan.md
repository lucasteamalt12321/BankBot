# Trivia Game Implementation Plan: Брейн-Ринг по Канону Олеговируса

Этот документ содержит пошаговый план и спецификацию викторины (Брейн-Ринг) по канону вселенной Олеговируса в Telegram-боте.

---

## 1. Функциональные требования

1. **Команда запуска:** Команда `/trivia` (с поддержкой mentioned-роутинга `/trivia@lt_lo_game_bot` для групп).
2. **Алгоритм выбора факта:**
   - Выбирает случайный факт из пула вопросов канона `bot/trivia/questions.py` (который синхронизирован с Google Doc).
   - Динамически выбирает 3 случайных правильных ответа из других вопросов в качестве неверных вариантов (дистракторов/фэйков).
   - Перемешивает варианты ответов при каждом запуске.
3. **Механика ответа (Telegram Poll):**
   - Викторина отправляется в виде нативного **неанонимного Telegram-опроса (Quiz)**.
   - Это позволяет игрокам голосовать прямо в интерфейсе Telegram.
   - Первому, кто ответит правильно, бот автоматически отправляет поздравительный ответ (reply) с начислением монет.
   - Опрос автоматически закрывается (`stop_poll`) после первого правильного ответа.
4. **Награда:** 25 монет за правильный ответ (начисляются напрямую в Supabase PostgreSQL с логированием транзакции).
5. **Защита от спама и блокировки:**
   - Ограничение: не более одной активной викторины в одном чате одновременно.
   - Таймаут на автозакрытие вопроса: если никто не ответил за 60 секунд, опрос закрывается, игра снимается с блокировки.

---

## 2. Архитектура и Структура файлов

Новый функционал оформлен в директории `bot/trivia/` и покрыт тестами:

1. `bot/trivia/questions.py` — пул вопросов по канону и генератор дистракторов `generate_trivia_question()`.
2. `bot/trivia/service.py` — сервис для начисления очков в БД и логирования транзакций `TriviaService`.
3. `bot/trivia/commands.py` — команда `/trivia` и обработчик результатов голосов `PollAnswerHandler`.
4. `tests/unit/test_trivia_game.py` — автоматические тесты для проверки генератора и начисления балансов в БД.
