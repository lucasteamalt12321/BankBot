# Trivia Game Implementation Plan: Брейн-Ринг по Канону Олеговируса

Этот документ содержит пошаговый план внедрения интерактивной викторины (Брейн-Ринг) по канону вселенной Олеговируса в Telegram-бот.

---

## 1. Функциональные требования

1. **Команда запуска:** Команда `/trivia` (с поддержкой mentioned-роутинга `/trivia@lt_lo_game_bot` для групп).
2. **Пул вопросов:** 15-20 детализированных вопросов, основанных на канонической базе знаний `bot/ai/knowledge.py` (Олеговирус, LTL-паразит, Чайная религия, Nine Circles, LTRS, сахарная экономика).
3. **Механика ответа:**
   - Сообщение с вопросом отправляется в чат с inline-кнопками (вариантами ответов).
   - Варианты перемешиваются при каждом запуске.
   - Первому, кто нажал на правильную кнопку, мгновенно начисляются монеты.
   - Бот редактирует исходное сообщение: показывает правильный ответ, имя победителя и закрывает кнопки.
4. **Награда:** 25 монет за правильный ответ (начисляются напрямую в Supabase PostgreSQL с логированием транзакции).
5. **Защита от спама и блокировки:**
   - Ограничение: не более одной активной игры в одном чате одновременно.
   - Таймаут на автозакрытие вопроса: если никто не ответил за 60 секунд, кнопки удаляются, игра закрывается.

---

## 2. Архитектура и Структура файлов

Новый функционал будет оформлен в виде изолированного модуля в директории `bot/trivia/`:

1. `bot/trivia/questions.py` — пул канонических вопросов и вариантов ответов.
2. `bot/trivia/service.py` — сервис для управления состоянием викторины, проверки ответов и начисления очков в БД.
3. `bot/trivia/commands.py` — Telegram callback handlers для команды `/trivia` и callback-кнопок.
4. `tests/unit/test_trivia_game.py` — автоматические тесты для проверки выбора вопросов, проверки ответов и начисления очков.

---

## 3. Пошаговый план реализации

### Этап 1: Данные и Базовый сервис (Trivia Service)
1. Создать `bot/trivia/questions.py` с пулом вопросов. Каждый вопрос содержит:
   - `id`: уникальный идентификатор.
   - `text`: текст вопроса.
   - `options`: список вариантов ответов.
   - `correct_index`: индекс правильного ответа.
   - `explanation`: краткое каноническое объяснение.
2. Создать `bot/trivia/service.py` с классом `TriviaService`:
   - Метод `get_random_question()` для выбора случайного вопроса.
   - Метод `process_answer(chat_id, user_id, game_id, selected_index)`:
     - Проверяет активность игры.
     - Проверяет правильность ответа.
     - Начисляет монеты победителю через `SessionLocal` и логирует транзакцию `trivia_win`.

### Этап 2: Telegram Команды и Кнопки
1. Реализовать handler команды `/trivia` в `bot/trivia/commands.py`:
   - Проверяет блокировку чата.
   - Выбирает случайный вопрос.
   - Отправляет сообщение с перемешанными кнопками.
   - Регистрирует `game_id` во временном in-memory реестре активных игр бота.
2. Реализовать `CallbackQueryHandler` с фильтром `trivia:`:
   - Парсит `trivia:<game_id>:<question_id>:<selected_index>`.
   - Запрашивает `TriviaService` для валидации.
   - При успехе: редактирует сообщение (показывает «Победитель: @username» и объяснение), начисляет награду и удаляет игру из реестра.
   - При ошибке (неправильный ответ): отправляет всплывающее уведомление (alert) игроку «Неверно! Попробуйте другие варианты».

### Этап 3: Подключение и Тестирование
1. Подключить `trivia_command` и `trivia_callback_handler` в `bot/bot.py`.
2. Зарегистрировать команду в общем хелпе и списке `/commands`.
3. Написать unit-тесты `tests/unit/test_trivia_game.py`:
   - Тест на случайный выбор вопросов.
   - Тест на успешное начисление монет при правильном ответе.
   - Тест на блокировку дублирующихся игр.
4. Запустить `ruff` и `py_compile`.
5. Деплой на Vercel Production и ручная проверка в Telegram-группе.
