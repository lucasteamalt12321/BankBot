# ТАБУ - Что НЕЛЬЗЯ делать в проекте

## 🚫 Критические запреты

### 1. НЕ создавать лишние файлы в корне проекта

**ЗАПРЕЩЕНО:**
- Тестовые файлы в корне (`test.db`, `test_*.db`)
- Временные файлы и скрипты
- Дублирующие конфигурации
- Случайные заметки и документы

**ПРАВИЛЬНО:**
- Тесты → `tests/`
- Скрипты → `scripts/`
- Конфигурация → `config/`
- Документация → `docs/`
- Временные файлы → `.gitignore`

**В корне должны быть ТОЛЬКО:**
- `README.md` - главная документация
- `run_bot.py` - точка входа
- `.gitignore` - правила игнорирования
- Конфигурационные файлы уровня проекта (если необходимо)

---

### 2. НЕ создавать копии файлов в разных папках

**ЗАПРЕЩЕНО:**
- `bot.db` в корне И `data/bot.db`
- `config.py` в нескольких местах
- Дублирование парсеров
- Копии одних и тех же утилит

**ПРАВИЛЬНО:**
- Одна база данных: `data/bot.db`
- Одна конфигурация: `config/` или `src/config.py`
- Один модуль - одно место
- Используй импорты, а не копирование

**Если нужна копия:**
- Создай символическую ссылку
- Используй импорт модуля
- Вынеси общий код в shared модуль

---

### 3. НЕ дублировать базы данных

**ОСОБЕННО КРИТИЧНО для БД!**

**ЗАПРЕЩЕНО:**
- Несколько `bot.db` в разных местах
- Тестовые БД в корне
- Бэкапы вне папки `backups/`
- Копирование БД вручную

**ПРАВИЛЬНО:**
- Рабочая БД: `data/bot.db`
- Тестовые БД: `tests/*.db` (временные, в .gitignore)
- Бэкапы: `backups/bot_backup_YYYYMMDD_HHMMSS.db`
- Используй скрипт `scripts/merge_databases.py` для объединения

---

### 4. НЕ создавать временные файлы без очистки

**ЗАПРЕЩЕНО:**
- Оставлять `temp.py`, `test123.py`
- Создавать `old_`, `backup_`, `copy_` файлы
- Коммитить `.pyc`, `__pycache__`

**ПРАВИЛЬНО:**
- Используй `.gitignore`
- Удаляй временные файлы после использования
- Для экспериментов создай отдельную ветку

---

### 5. НЕ хардкодить пути и ID

**ЗАПРЕЩЕНО:**
```python
db_path = "C:/Users/admin/bot.db"  # ❌
admin_id = 123456789  # ❌
```

**ПРАВИЛЬНО:**
```python
db_path = Path("data/bot.db")  # ✅
admin_id = config.get_admin_id()  # ✅
```

## ✅ Правила организации проекта

### Структура папок:
```
BankBot/
├── bot/              # Telegram bot код
├── core/             # Бизнес-логика
├── database/         # Работа с БД
├── src/              # Общие утилиты
├── config/           # Конфигурация
├── data/             # Данные (БД, JSON)
├── scripts/          # Вспомогательные скрипты
├── tests/            # Все тесты
├── docs/             # Документация
├── backups/          # Бэкапы БД
├── for_programmer/   # Инструкции для разработчиков
└── README.md         # Главная документация
```

### Правило одного места:
- Один тип файлов - одна папка
- Один модуль - одна ответственность
- Одна база данных - одно место

### Перед коммитом проверь:
```bash
# Нет ли лишних файлов в корне?
ls -la | grep -E "test|temp|old|backup|copy"

# Нет ли дублей БД?
find . -name "*.db" -not -path "./backups/*" -not -path "./tests/*"

# Нет ли дублей кода?
find . -name "*.py" | xargs -I {} basename {} | sort | uniq -d
```

## ⚠️ Что делать, если нарушил правила?

1. **Найди дубли:**
   ```bash
   python scripts/merge_databases.py  # для БД
   ```

2. **Удали лишнее:**
   ```bash
   # Переместить в правильную папку
   mv test_*.db tests/
   
   # Или удалить
   rm temp_*.py
   ```

3. **Обнови .gitignore:**
   ```
   # Временные файлы
   temp_*
   test_*
   old_*
   backup_*
   ```

4. **Почисть git:**
   ```bash
   git rm --cached bot.db  # если случайно закоммитил
   ```
