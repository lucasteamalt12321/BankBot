# \# Система Memory Bank

# \# Ты действуешь как инженер, который не обладает долгосрочной памятью. Твой единственный устойчивый контекст — директория `/memory\\\_bank`. Поддерживай документацию в актуальном состоянии, синхронизируя её с кодом.

# \# 1\\. Фундаментальные правила (Policy)

# 

# \# `docs/README.md` — единственный источник информации о высокоуровневой архитектуре.

# 

# \# Канонический источник процента выполнения: `memory\\\_bank/projectbrief.md`.

# 

# \# В `memory\\\_bank/projectbrief.md` обязателен раздел `## Project Deliverables`.

# 

# \# Создать раздел автоматически, если он отсутствует, при любой задаче, связанной с прогрессом, планированием или синхронизацией memory\\\_bank.

# 

# \# `## Project Deliverables` — канонический список результатов, по которому считается процент выполнения. Не журнал сессии.

# 

# \# Каждый deliverable обязан иметь: стабильный ID, краткое название, статус (`pending` / `in\\\_progress` / `completed` / `blocked`), вес в процентах.

# 

# \# Сумма весов всех deliverables = `100`.

# 

# \# Процент выполнения считается только по `## Project Deliverables`.

# 

# \# `activeContext.md`, `progress.md`, `Changelog` — операционный контекст и история, не источник процента.

# 

# \# Каждый новый пункт ТЗ или scope change добавлять в `## Project Deliverables` до или одновременно с реализацией.

# 

# \# После каждой задачи, влияющей на готовность — обновить статусы deliverables.

# 

# \# Если deliverables не синхронизированы с кодом — сначала восстановить, потом считать процент. Иначе процент = `нет достоверных данных`.

# 

# \# 2\\. Память проекта

# 

# \# Перед началом любой работы перечитай `/memory\\\_bank`, особенно `activeContext.md` и `progress.md`.

# 

# \# `/memory\\\_bank` — твоя операционная система. Без актуализации теряешь контекст.

# 

# \# Любые изменения в архитектуре, маршрутах или модулях отражать в `docs/README.md`.

# 

# \# 3\\. Контроль изменений

# \# `progress.md` всегда содержит `last\\\_checked\\\_commit` (хеш или дата).

# 

# \# При каждом сеансе сравнивай: `git log <last\\\_checked\\\_commit>...`

# 

# \# При изменении документации — обновить хеш.

# \# Обязательные файлы в `/memory\\\_bank`:
