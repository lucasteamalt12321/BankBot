# План разработки проекта

## Фаза 1 — Основа (Приоритет: критический)

1. Вынести все конфиденциальные данные (токены, ID, пароли) в переменные окружения
2. Создать централизованную конфигурацию на базе Pydantic Settings с поддержкой окружений (dev/prod/test)
3. Закрепить версии зависимостей, разделить на `requirements.txt` и `requirements-dev.txt`
4. Исправить все импорты после рефакторинга конфигурации

## Фаза 2 — Архитектура (Приоритет: высокий)

5. Реализовать слой репозиториев для всех операций с БД, убрать прямые SQL запросы
6. Создать service layer, вынести бизнес-логику из handlers
7. Разбить монолитный `bot.py` на модули по группам команд
8. Настроить централизованный middleware для обработки ошибок с уведомлением администратора
9. Реализовать graceful shutdown с корректным закрытием ресурсов
10. Перенести конфигурацию парсинга из JSON файлов в БД, создать ParserRegistry

## Фаза 3 — Безопасность и качество (Приоритет: средний)

11. Добавить блокировки для операций с балансом, реализовать Unit of Work для атомарных транзакций
12. Настроить connection pooling
13. Провести аудит SQL запросов на SQL injection, добавить параметризацию везде где её нет
14. Реализовать систему алиасов пользователей с fallback на имя
15. Настроить DI контейнер зависимостей

## Фаза 4 — Очистка (Приоритет: низкий)

16. Провести аудит неиспользуемого кода (D&D система, система кланов, неиспользуемые таблицы БД)
17. Удалить или задокументировать deprecated функции
18. Объединить дублирующиеся парсеры

## Фаза 5 — Тесты и документация (Приоритет: низкий)

19. Написать E2E тесты для основных сценариев (регистрация, начисление очков, покупки)
20. Добавить тесты на SQL injection, race conditions, graceful shutdown
21. Довести coverage до 80%+
22. Обновить README, создать DEPLOYMENT.md, архитектурные диаграммы
23. Добавить docstrings (Google style) во все публичные методы
