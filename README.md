# BankBot - парсинг игровой активности, Telegram-банк и мост TG -> VK

Репозиторий содержит основной Telegram-бот для парсинга игровой активности и начисления банковских монет, а также банково-админскую платформу вокруг этого контура и bridge-компоненты для пересылки сообщений между Telegram и VK.

Ключевая продуктовая цель проекта — **работающий production-grade парсинг игровых сообщений**. При этом текущая реализация честно разделена на два слоя:

- **основной готовый контур**: банк, пользователи, балансы, магазин, админка, feedback, AI-lite, режимы ответов, PostgreSQL/HF runtime;
- **главный незакрытый контур**: стабильный автоматический парсинг игровых сообщений. Сейчас ручной/инфраструктурный парсинг присутствует, но end-to-end production-парсинг требует отдельной стабилизации и остаётся главным приоритетом развития.

## Что есть в проекте

| Компонент | Назначение |
|-----------|------------|
| `BankBot` | Основной Telegram-бот: целевой парсинг игровой активности, баланс, магазин, админ-команды, feedback, AI-lite и режимы ответов |
| `BridgeBot` | Пересылка сообщений и медиа из Telegram в VK |
| `VK Bot` | Отдельная точка входа для VK Long Poll и публикации в VK |

## Основные возможности BankBot

- целевой production-grade парсинг игровых сообщений и начисление монет за активность;
- банковые монеты, баланс, история транзакций и профиль пользователя;
- магазин, покупки, инвентарь и связь с администратором;
- игровые и D&D-команды, достижения, уведомления и социальные функции;
- админ-панель, начисление/списание монет, рассылка `/broadcast`, списки пользователей и диагностика;
- `/feedback`, `/suggest`, `/complaint`, `/feedback_list` и защищённый внешний reader `/feedback`;
- бесплатный локальный AI-lite помощник: `/ai`, `/ask`, `/ai_help`, `/ai_update_knowledge`;
- persistent PostgreSQL/Supabase storage в production/Hugging Face, SQLite как local/dev fallback;
- Hugging Face health/log endpoints: `/health`, `/logs`, `/metrics`.

## Статус парсинга

Парсинг — самое важное направление проекта и причина существования BankBot как игрового банка. Но текущий статус нельзя описывать как полностью готовый автоматический парсер:

- инфраструктура парсинга, правила и ручной контур есть;
- автоматический production-парсинг требует стабилизации на реальных игровых сообщениях;
- банк, PostgreSQL, админка, feedback и режимы ответов были приоритетно усилены как фундамент, без которого результаты парсинга нельзя надёжно хранить, проверять и администрировать;
- следующий продуктовый фокус после текущих bugfix-задач — вернуть парсинг в центр roadmap и довести его до устойчивого end-to-end сценария.

## Режимы ответов

BankBot поддерживает три режима ответа, которые выбираются отдельно для каждого Telegram-пользователя:

| Команда | Назначение |
|---------|------------|
| `/short` | Короткие ответы, безопасные для Hugging Face и групповых чатов |
| `/long` | Полные подробные ответы |
| `/watch` | Режим часов: сверхкороткие ответы и управление через шаблонные ответы |

Администратор может задать общий default для всех:

- `/short_all`
- `/long_all`
- `/watch_all`

После этого каждый пользователь всё равно может выбрать личный режим заново.

### Управление с часов

В режиме `/watch` бот использует только короткие сообщения и понимает 10 шаблонных ответов часов как быстрые действия:

| Ответ часов | Действие |
|-------------|----------|
| `ОК` | открыть профиль |
| `Да` | открыть админ-раздел |
| `Спасибо` | показать баланс |
| `Спасибо, нет` / `Спасибо нет` | открыть магазин |
| `Великолепно` | открыть игры |
| `Спасибо еще раз` | показать AI help |
| `Скоро увидимся` | показать список команд |
| `Скоро буду` | открыть уведомления |
| `Я занят(а)` | включить личный режим часов |
| `Нет` | отмена/подсказка по watch-действиям |

`Я занят(а)` включает `/watch` для конкретного человека даже без предварительной команды `/watch`.

## Диалоговый кодер шаблонов

В основном Telegram-боте есть модуль `/coder`, который кодирует смысл фразы через последовательность из тех же 10 фиксированных текстовых шаблонов:

1. ОК
2. Да
3. Спасибо
4. "Спасибо, нет"
5. Великолепно
6. Спасибо еще раз
7. Скоро увидимся
8. Скоро буду
9. Я занят(а)
10. Нет

Кодер использует параметрическое состояние (`topic`, `time_minutes`, `intensity`, `politeness`, `confidence`, `repeat_flag`, история шаблонов), а не таблицы пар/троек. Поэтому последовательность может быть любой длины: после каждого шаблона бот показывает текущий смысл, предварительный просмотр всех 10 следующих вариантов и сортирует их по силе изменения состояния.

Команды:

- `/coder` — открыть кодер и сбросить состояние;
- `/reset` — сбросить текущую последовательность;
- `/done` — завершить ввод и вывести финальную фразу;
- `/help` — показать инструкцию по шаблонам.

Отдельно от кодера есть краткий режим обычных меню. Он включён по умолчанию:

- `/short` — включить короткие ответы только для себя;
- `/long` — вернуть полные ответы только для себя;
- `/short_all` — включить короткие ответы для всех;
- `/long_all` — вернуть полные ответы для всех.

## Точки входа

> Важно: локально проект следует запускать через **Python 3.12**. Запуск через Python 3.14 может падать из-за несовместимости `python-telegram-bot==20.7` с runtime.

| Компонент | Запуск |
|-----------|--------|
| BankBot | `py -3.12 run_bot.py` |
| BankBot напрямую | `py -3.12 -m bot.main` |
| BridgeBot | `py -3.12 -m bridge_bot.main` |
| VK Bot | `py -3.12 -m vk_bot.main` |

## Быстрый старт

Если вы просто хотите проверить работу бота, проект клонировать не нужно: тестируйте актуальный production-бот прямо в Telegram-группе LucasTeam:

https://t.me/lucasteamgroup

Локальный запуск нужен только для разработки, отладки или внесения изменений в код.

### 1. Установка зависимостей

```powershell
py -3.12 -m pip install -r requirements.txt
```

### 2. Настройка окружения

```powershell
Copy-Item config/.env.shared.example config/.env.shared
Copy-Item config/.env.local.example config/.env.local
```

Схема env-файлов:

- `config/.env.shared` — коммитируемая безопасная часть: feature flags, logging, database defaults, proxy policy
- `config/.env.local` — локальная секретная часть: токены, админские ID, приватные bridge/VK значения

Минимально необходимые переменные в `config/.env.local`:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_TELEGRAM_ID=123456789
```

Базовые безопасные настройки в `config/.env.shared`:

```env
DATABASE_URL=sqlite:///data/bot.db
ENV=development
LOG_LEVEL=INFO
# При системном VPN (например, AmneziaVPN) оставляйте пустым
PROXY_URL=
```

Для bridge-режима дополнительно в `config/.env.local`:

```env
BRIDGE_ENABLED=false
BOT_TOKEN_BRIDGE=your_bridge_bot_token
TG_CHANNEL_ID=-1001234567890
VK_TOKEN=your_vk_token
VK_PEER_ID=2000000001
```

### Быстро с нуля: 3 команды

```powershell
Copy-Item config/.env.shared.example config/.env.shared
Copy-Item config/.env.local.example config/.env.local
py -3.12 run_bot.py
```

После этого открой `config/.env.local` и заполни минимум:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_TELEGRAM_ID=123456789
```

### 3. Подготовка данных

```powershell
New-Item -ItemType Directory -Force data, logs
py -3.12 database/initialize_system.py
```

### 4. Запуск

```powershell
py -3.12 run_bot.py
```

Подробный локальный сценарий запуска: `RUN.md`

## Архитектура

### Основные каталоги

| Путь | Назначение |
|------|------------|
| `bot/` | Основной runtime BankBot |
| `bridge_bot/` | TG -> VK bridge |
| `vk_bot/` | Независимый VK runtime |
| `bank_bot/` | Repository/service/middleware слои |
| `src/` | Канонический конфиг и startup-инфраструктура |
| `database/` | Подключение, schema sync, Alembic, bootstrap |
| `tests/` | Unit, integration и smoke-тесты |
| `memory_bank/` | Операционный контекст и прогресс проекта |

### Конфигурация

Канонический источник конфигурации: `src/config.py`.

Ключевые принципы:

- основной env-файл: `config/.env`
- bridge и vk config-модули используют compatibility-обёртки над `src.config`
- схема БД приводится к актуальному состоянию при запуске через `database/schema.py`
- production/Hugging Face использует `DATABASE_URL`/`POSTGRES_URL`/`SUPABASE_DB_URL`; SQLite предназначен для локальной разработки
