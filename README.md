# 🤖 Telegram-бот банк-аггрегатор LucasTeam

Мета-игровая платформа для автоматического отслеживания активности в играх и начисления банковских монет.

## 🚀 Быстрый старт

### Запуск бота
```bash
python run_bot.py
```

### Первое использование
1. Отправьте `/start` боту в Telegram
2. Получите полный список команд
3. Используйте `/balance` для проверки баланса
4. Начните играть - монеты начисляются автоматически!

## ⚙️ Технические особенности

### 🔧 Унифицированная система конфигурации

Проект использует единую систему конфигурации на основе Pydantic Settings:

- **Единый источник конфигурации**: `src/config.py` - все настройки в одном месте
- **Валидация настроек**: автоматическая проверка корректности значений при запуске
- **Поддержка окружений**: development, test, staging, production
- **Переменные окружения**: все настройки можно задать через environment variables
- **Безопасность**: конфиденциальные данные хранятся в `.env` файлах (не в коде)

**Использование в коде:**
```python
from src.config import settings

# Доступ к настройкам
bot_token = settings.BOT_TOKEN
admin_id = settings.ADMIN_TELEGRAM_ID
db_url = settings.DATABASE_URL
```

**Поддержка множественных окружений:**

Создайте отдельные файлы конфигурации для каждого окружения:
```
config/
├── .env                    # Общие настройки
├── .env.development        # Разработка
├── .env.test               # Тестирование
├── .env.production         # Продакшн
```

Запуск в разных окружениях:
```bash
# Разработка
ENV=development python run_bot.py

# Продакшн
ENV=production python run_bot.py
```

Подробнее: [docs/MULTIPLE_ENVIRONMENTS.md](docs/MULTIPLE_ENVIRONMENTS.md)

### Устранение типичных проблем

#### ❌ Ошибка: "❌ .env file not found"

**Причина:** Файл `config/.env` не создан.

**Решение:**
```bash
cp config/.env.example config/.env
# Отредактируйте config/.env и укажите ваши данные
```

#### ❌ Ошибка: "BOT_TOKEN cannot be empty"

**Причина:** В `config/.env` не указан токен бота или указан неверно.

**Решение:** Проверьте, что в `config/.env` есть строка:
```env
BOT_TOKEN=ваш_реальный_токен_от_BotFather
```

#### ❌ Ошибка: "ADMIN_TELEGRAM_ID must be a positive integer"

**Причина:** Telegram ID администратора не указан или указан неверно.

**Решение:** Получите ваш ID у [@userinfobot](https://t.me/userinfobot) и укажите в `config/.env`:
```env
ADMIN_TELEGRAM_ID=123456789
```

#### ❌ Бот не отвечает на команды

**Возможные причины и решения:**
1. **Неверный токен** - проверьте `BOT_TOKEN` в `config/.env`
2. **Бот не запущен** - убедитесь, что `python run_bot.py` выполняется без ошибок
3. **Проблемы с сетью** - проверьте интернет-соединение
4. **База данных не инициализирована** - выполните `python database/initialize_system.py`

#### ❌ Ошибка при инициализации базы данных

**Решение:**
```bash
# Удалите старую базу данных (если есть)
rm data/bot.db

# Создайте заново
python database/initialize_system.py
```

## 🔐 Environment Variables

Все настройки бота управляются через переменные окружения в файле `config/.env`. Это обеспечивает безопасность и гибкость конфигурации.

### Обязательные переменные

Эти переменные **ДОЛЖНЫ** быть установлены для запуска бота:

| Переменная | Описание | Пример |
|------------|----------|--------|
| `BOT_TOKEN` | Токен Telegram бота от [@BotFather](https://t.me/BotFather) | `1234567890:ABCdefGHI...` |
| `ADMIN_TELEGRAM_ID` | Telegram ID администратора (от [@userinfobot](https://t.me/userinfobot)) | `123456789` |
| `DATABASE_URL` | URL подключения к базе данных | `sqlite:///data/bot.db` |

### Опциональные переменные

Эти переменные имеют значения по умолчанию и могут быть опущены:

#### Окружение и режим работы

| Переменная | Описание | По умолчанию | Возможные значения |
|------------|----------|--------------|-------------------|
| `ENV` | Режим окружения | `development` | `development`, `test`, `staging`, `production` |
| `DEBUG` | Режим отладки (автоматически включается в development) | `false` | `true`, `false` |
| `TEST_MODE` | Тестовый режим (автоматически включается в test) | `false` | `true`, `false` |

#### Парсинг игровых результатов

| Переменная | Описание | По умолчанию |

### Настройка файла .env

1. Скопируйте файл-пример:
   ```bash
   cp config/.env.example config/.env
   ```

2. Отредактируйте `config/.env` и установите обязательные переменные

3. (Опционально) Настройте дополнительные параметры под ваши нужды

4. Проверьте конфигурацию:
   ```bash
   python -c "from src.config import settings; print('✅ Configuration valid')"
   ```

### Множественные окружения

Вы можете создать отдельные файлы конфигурации для разных окружений:

```
config/
├── .env                    # Общие настройки
├── .env.development        # Разработка
├── .env.test               # Тестирование
├── .env.staging            # Предпродакшн
├── .env.production         # Продакшн
```

Запуск в конкретном окружении:
```bash
ENV=production python run_bot.py
```

Подробнее: [docs/MULTIPLE_ENVIRONMENTS.md](docs/MULTIPLE_ENVIRONMENTS.md)

### Безопасность

⚠️ **Важно:**
- **Никогда** не коммитьте файл `.env` в систему контроля версий
- Файл `.env` уже добавлен в `.gitignore`
- Используйте `.env.example` как шаблон без реальных секретов
- В продакшн окружении используйте переменные окружения системы или секрет-менеджеры
- Регулярно ротируйте `BOT_TOKEN` и другие секреты

### Валидация при запуске

Бот автоматически проверяет конфигурацию при запуске:
- ✅ Проверка существования файла `.env`
- ✅ Валидация обязательных переменных
- ✅ Проверка формата значений
- ✅ Тестирование подключения к базе данных

Если конфигурация невалидна, бот не запустится и выведет понятное сообщение об ошибке с инструкциями по исправлению.

### Система парсинга
Автоматически отслеживает сообщения из игр и начисляет монеты согласно настроенным правилам парсинга. 

**Поддерживаемые форматы:**
- 🃏 GD Cards - парсинг новых карт и начисление очков
- 🎣 Shmalala - парсинг рыбалки и начисление монет

**Использование:**
```python
from core.parsers.simple_parser import parse_game_message

result = parse_game_message(message_text)
if result:
    print(f"Тип: {result['type']}")
    print(f"Пользователь: {result['user']}")
    print(f"Количество: {result['amount']}")
```

Подробнее: [docs/PARSER_INTEGRATION.md](docs/PARSER_INTEGRATION.md)

> ⚠️ **Внимание:** Система парсинга находится в тестовом режиме и может требовать дополнительной настройки для корректной работы с конкретными играми.
